## Постановка задачи

Необходимо реализовать иерархию классов с абстрактным базовым классом `IterableObject`, который содержит чисто виртуальные функции `length()` (возвращает длину/размер контейнера) и `contains(const std::string& key)` (проверяет наличие ключа в объекте). Требуется создать:

- Класс `String`, представляющий строку в стиле C с динамическим выделением памяти (реализовать правило трёх).
- Шаблонный класс `List<T>`, реализующий динамический массив элементов типа `T` (ручное управление памятью, автоматическое расширение, запрет копирования).
- Класс `StringList`, наследующий от `String` и `List<String>` (множественное наследование), с собственной логикой `contains`: проверка наличия в списке строки, равной ключу или содержащей его как подстроку.
- В функции `main` реализовать консольное меню для создания объектов, их отображения, вызова `contains`, удаления и тестирования полиморфизма через указатели на `IterableObject`.
- Обеспечить отсутствие утечек памяти (проверка через valgrind).


## Реализация

В проекте реализованы следующие компоненты:

- **IterableObject.h** – абстрактный базовый класс с виртуальным деструктором, чисто виртуальными методами `length()` и `contains(const std::string&)`.  
- **String.h / String.cpp** – класс строки с ручным управлением памятью: конструкторы (из C-строки и `std::string`), конструктор копирования, оператор присваивания, деструктор, переопределение `length()` и `contains()`, оператор `==` для сравнения строк.  
- **List.h / List.cpp** – шаблонный класс динамического массива. Реализован с явным инстанцированием для типов `int`, `double`, `std::string` и `String`. Содержит методы `add`, `get`, `size`, переопределяет `length()` (возвращает размер) и `contains()` (ищет элемент, равный преобразованному из ключа значению). Использует вспомогательную функцию `parseFromString<T>` со специализациями.  
- **StringList.h / StringList.cpp** – класс с множественным наследованием от `String` и `List<String>`. Переопределяет `contains()` (поиск в списке с учётом равенства или вхождения подстроки) и `length()` (возвращает размер списка). Конструктор инициализирует базовую строку.  
- **main.cpp** – тестовая программа с консольным меню: создание объектов `String`, `List<int>`, `List<double>`, `List<std::string>`, `StringList`; просмотр всех объектов с определением их типа через `dynamic_cast`; вызов `contains` для конкретного объекта; запуск теста `contains` для всех объектов через указатели на `IterableObject`; удаление объекта.  
- **Makefile** – сборка проекта, очистка, запуск под valgrind.

Для полиморфизма используется виртуальное наследование от `IterableObject` в `String` и `List<T>`, чтобы избежать неоднозначности в `StringList`. Все динамические ресурсы освобождаются корректно (правило трёх для `String`, запрет копирования для `List`, виртуальный деструктор в базовом классе).


## Запуск

Для сборки проекта используется Makefile:

- Сборка проекта:
  ```
  make
  ```
  Будет создан исполняемый файл `test`.

- Запуск программы:
  ```
  ./test
  ```

- Проверка утечек памяти с помощью valgrind:
  ```
  valgrind --leak-check=full --show-leak-kinds=all ./test
  ```

- Очистка скомпилированных файлов:
  ```
  make clean
  ```
