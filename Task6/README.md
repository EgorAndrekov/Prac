# Task6

Данный проект демонстрирует взаимодействие нескольких процессов при помощи системных семафоров System V на базе модели регулируемого пешеходного перехода.

## Описание задачи

Реализована система "Светофор-Пешеходы", где:
1. Один процесс имитирует светофор, который поочередно горит красным (5 секунд) и зеленым (3 секунды) светом.
2. Несколько процессов-пешеходов периодически пытаются перейти дорогу, соблюдая правила дорожного движения.
3. Пешеходы сообщают о своих действиях в поток stderr, указывая свой PID и текущее действие.
4. Пешеходы запускаются с параметром, определяющим частоту попыток перехода.
5. Реализована программа демонтажа светофора, которая корректно завершает все процессы.

## Состав решения

### 1. Светофор (`l` - light)
Процесс, создающий и управляющий системными семафорами:
- Создает набор из двух семафоров по ключу `0xDEADBEEF`:
  - `SEM_RED` (индекс 0): 1 = красный свет, 0 = зеленый свет
  - `SEM_ACTIVE` (индекс 1): 1 = светофор активен, 0 = сигнал демонтажа
- Циклически переключает цвета:
  - Красный свет: 5 секунд (устанавливает `SEM_RED = 1`)
  - Зеленый свет: 3 секунды (устанавливает `SEM_RED = 0`)
- Регулярно проверяет флаг `SEM_ACTIVE` и наличие семафора
- Корректно завершается при получении сигнала демонтажа

### 2. Пешеход (`p` - pedestrian)
Процесс-пешеход с поведением:
- Запускается с аргументом `N` - средний интервал между попытками перехода (в секундах)
- Цикл поведения:
  1. **Гуляет**: заданное в аргументах программы при запуске время
  2. **Ждет зеленого сигнала**: проверяет семафор `SEM_RED`
  3. **Переходит дорогу**: когда `SEM_RED = 0` (зеленый свет)
- Все действия логируются в stderr с указанием PID процесса
- Периодически проверяет флаг `SEM_ACTIVE`
- Корректно завершается при демонтаже светофора

### 3. Демонтаж (`r` - remove)
Программа корректного завершения системы:
1. Устанавливает флаг `SEM_ACTIVE = 0` (сигнал завершения)
2. Дает время светофору и пешеходам на корректное завершение
3. Удаляет набор семафоров

## Технические особенности

### Используемые технологии
- System V IPC семафоры
- Межпроцессное взаимодействие
- Обработка ошибок `EIDRM` (удаленный семафор)

### Сборка
```bash
make        # сборка всех программ
make clean  # удаление исполняемых файлов
```

Создаются исполняемые файлы:
- `l` (light) - светофор
- `p` (pedestrian) - пешеход
- `r` (remove) - демонтаж

### Запуск (пример)

1. **Запуск светофора:**
```bash
./l &
```

2. **Запуск нескольких пешеходов:**
```bash
./p 5 &  # пытается переходить каждые ~5 секунд
./p 7 &  # пытается переходить каждые ~7 секунд
./p 10 & # пытается переходить каждые ~10 секунд
```

3. **Демонтаж системы:**
```bash
./r
```

### Пример вывода пешехода
```

Пешеход 109674: Ждет зеленого сигнала
Пешеход 109674: Переходит дорогу по переходу
Пешеход 109674: перешел дорогу
Пешеход 109674: светофор демонтирован, завершение
```

```

Семафор создан с ID: 3
КРАСНЫЙ свет (5 секунд)
ЗЕЛЕНЫЙ свет (3 секунды)
КРАСНЫЙ свет (5 секунд)

```

## Примечания

1. **Порядок запуска:** Сначала должен быть запущен светофор (`./l`), затем пешеходы.
2. **Множественные пешеходы:** Можно запускать любое количество процессов-пешеходов.
3. **Корректное завершение:** Программа демонтажа гарантирует корректное завершение всех процессов.
4. **Время ожидания:** При демонтаже дается 5 секунд на завершение работы светофора.

## Алгоритм работы

1. Светофор создает семафоры и начинает цикл переключения цветов.
2. Пешеходы подключаются к существующим семафорам.
3. При переходе пешеход ждет, когда `SEM_RED = 0` (зеленый свет).
4. При запуске демонтажа:
   - Устанавливается `SEM_ACTIVE = 0`
   - Светофор завершает работу при следующей проверке
   - Пешеходы завершаются при следующей проверке активности
   - Удаляются семафоры
